==========================================
BLOOM API - FUTURE ENHANCEMENT IDEAS
==========================================

TIER 1: QUICK WINS (1-2 days)
=============================

1. BLOOM PRESETS SYSTEM
   Predefined parameter sets:
   - SUBTLE: threshold=0.95, intensity=0.5
   - NORMAL: threshold=0.8, intensity=1.5
   - INTENSE: threshold=0.5, intensity=2.5
   - CINEMATIC: threshold=0.7, intensity=2.0, custom curve
   
   API Extension:
   enum class BloomPreset { Subtle, Normal, Intense, Cinematic };
   void SetPreset(BloomPreset preset);

2. BLOOM ACCUMULATION
   Temporal smoothing across frames:
   - Store previous frame bloom result
   - Blend with current frame (85% prev + 15% current)
   - Reduces flicker, smoother appearance
   - Memory cost: +1 texture (3-12 MB)

3. DYNAMIC INTENSITY CONTROL
   Real-time parameter adjustment:
   - Bind to input system
   - Lerp between states smoothly
   - Useful for gameplay events (explosions, etc.)

4. BLOOM VISUALIZATION DEBUG MODE
   Development tool:
   - Display bloom contribution only
   - Show threshold extraction mask
   - Visualize blur stages
   - Invaluable for tuning parameters

TIER 2: MEDIUM EFFORT (3-5 days)
=================================

5. COMPUTE SHADER IMPLEMENTATION
   GPU-optimized alternative:
   - Rewrite blur as compute shader
   - Better cache efficiency
   - Potential 2-3x speedup
   - Requires SM5.0+ GPU
   
   Code structure:
   #include "BloomComputeShaders.hlsl"
   class BloomEffectCompute : public BloomEffect

6. BLOOM PYRAMID (Multi-scale bloom)
   Hierarchical blur:
   - Generate downsampled bloom pyramid
   - Apply blur at each level
   - Upsample back to full resolution
   - Better quality-to-performance ratio
   - Suitable for very large blur radii

7. HDR SUPPORT
   Extended dynamic range:
   - Use R16F_G16F_B16F_A16F format
   - Preserve high dynamic range colors
   - Better for physically-based bloom
   - More realistic light propagation

8. BLOOM UPSAMPLING
   Quality improvement pass:
   - Upsampled bloom blended with source
   - Smooth transition between blur scales
   - Better edge preservation
   - Film-like appearance

TIER 3: ADVANCED (5-10 days)
=============================

9. BLOOM TINT BLENDING
   Color-space aware blending:
   - RGB vs HSV vs LCh
   - Per-channel intensity control
   - Separate R/G/B blur radii
   - Chromatic aberration simulation

10. LENS SIMULATION
    Realistic optical effects:
    - Bloom lens shape (star patterns)
    - Lens diffraction effects
    - Aperture shape influence
    - Bloom direction based on light position

11. BLOOM MOTION BLUR INTEGRATION
    Combined effects:
    - Directional bloom based on motion
    - Velocity-based blur radius
    - Integration with post-processing
    - Enhanced cinematic feel

12. BLOOM BUFFER POOLING
    Memory efficiency:
    - Maintain pool of texture resources
    - Reduce allocation/deallocation
    - Automatic sizing based on usage
    - Thread-safe resource access

TIER 4: RESEARCH (10+ days)
============================

13. AI-DRIVEN QUALITY SCALING
    Machine learning optimization:
    - Profile GPU cost per frame
    - ML model predicts optimal quality
    - Automatic adjustment for target FPS
    - Learns from frame history

14. BLOOM PREDICTION
    Anticipatory rendering:
    - Predict bloom needs next frame
    - Pre-compute likely bloom targets
    - Reduce latency perception
    - Improve responsiveness

15. CROSS-SCREEN BLOOM
    Multi-display effects:
    - Bloom propagation across monitors
    - Unified effect for ultrawide setups
    - Resolution-independent blooming
    - Lighthouse-like effects

16. REAL-TIME KERNEL GENERATION
    Procedural blur kernels:
    - User-defined kernel shapes
    - Organic bloom patterns
    - Artistic control
    - Per-frame kernel animation

17. BLOOM LIGHT TRACKING
    Smart bloom placement:
    - Detect light sources in scene
    - Apply bloom only near lights
    - Automatic bloom masking
    - Physically-informed rendering

18. VOLUMETRIC BLOOM
    3D bloom effect:
    - Bloom extends into 3D space
    - Affects particle systems
    - Godray-like appearance
    - Immersive visual depth

INTEGRATION OPPORTUNITIES:
==========================

WITH TONEMAPPING:
  - Coordinate bloom intensity with exposure
  - Bloom should be last post-process
  - Preserve HDR values before tonemapping

WITH ANTIALIASING:
  - Apply bloom before MSAA resolve
  - Consider bloom in TAA calculation
  - Temporal stability improvements

WITH MOTION BLUR:
  - Bloom/motion blur interaction
  - Combined effect profiling
  - Order-dependent results

WITH COLOR GRADING:
  - Color grade applied to bloom?
  - Or bloom applied to graded image?
  - LUT integration considerations

SHADER ENHANCEMENT IDEAS:
=========================

1. CUSTOM BLUR KERNELS
   Replace fixed 7-tap Gaussian:
   - Configurable kernel size
   - Different kernel shapes (Tent, Box, etc.)
   - Separable kernel support
   - Custom kernel uploading

2. BLOOM ANIMATION
   Frame-to-frame variation:
   - Oscillating intensity
   - Rotating bloom direction
   - Pulsating effect
   - Data-driven timing

3. DEPTH-BASED BLOOM
   Depth-aware effects:
   - Bloom only affects certain depths
   - Layered bloom compositing
   - Focus simulation
   - Depth-based masking

4. NORMAL MAP BLOOM
   Surface-dependent bloom:
   - Smooth surfaces bloom more
   - Rough surfaces bloom less
   - Surface tension visualization
   - Material-specific blooming
