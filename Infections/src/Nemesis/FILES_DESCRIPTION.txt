=== Asm ===
Asm\AsmFactory.h — AsmFactory.h — Класс AsmFactory — фабрика, создающая подходящий генератор машинного кода (AsmHelper32 или AsmHelper64) в зависимости от желаемой архитектуры (eAsmArch), типа модуля (eModType) или признака wow64-процесса.
Asm\AsmHelper32.h — AsmHelper32.h — Класс AsmHelper32 (наследник IAsmHelper) — генератор x86-кода для стабовых функций: формирует пролог/эпилог функции, вызов GenCall с поддержкой конвенций cdecl/stdcall/thiscall/fastcall, завершение потока с сохранением eax (ExitThreadWithStatus) и сигнализацию события с возвратом результата.
Asm\AsmHelper64.h — AsmHelper64.h — Класс AsmHelper64 (наследник IAsmHelper) — аналогичный генератор машинного кода для x86_64: формирует shadow-стек для вызовов по соглашению Windows x64, поддерживает переключение режима (EnableX64CallStack) и аналогичные методы ExitThreadWithStatus/SaveRetValAndSignalEvent.
Asm\IAsmHelper.h — IAsmHelper.h — Базовый интерфейс IAsmHelper на основе AsmJit X86Assembler; определяет общий контракт генерации пролога/эпилога/вызова и реализует переключение процессора между режимами x86 (SwitchTo86) и long mode x64 (SwitchTo64) через far ret трюки.
Asm\LDasm.h — LDasm.h — Заголовок легковесного длино-определяющего дизассемблера (length disassembler) LDasm: объявляет структуру ldasm_data и функции ldasm/SizeOfProc/ResolveJmp для определения длины x86/x64-инструкции и разрешения JMP-переходников.
Asm\AsmStack.hpp — AsmStack.hpp — Класс AsmStackAllocator — помощник распределения локальных переменных на стеке генерируемого ассемблерного кода (AllocVar/AllocArray), вычисляет смещения относительно rsp/rbp в зависимости от архитектуры x86/x64.
Asm\AsmVariant.hpp — AsmVariant.hpp — Структура AsmVariant — типобезопасная обёртка произвольного аргумента (число, указатель, строка, структура по значению) для передачи в генератор вызовов GenCall, автоматически определяет тип значения (imm/dataPtr/dataStruct/mem и т.д.) через шаблонный конструктор.
Asm\AsmHelper32.cpp — AsmHelper32.cpp — Реализация AsmHelper32: генерация x86 пролога/эпилога (push ebp/mov ebp,esp), формирование вызова GenCall с передачей аргументов через регистры (ecx/edx для thiscall/fastcall) и стек, вызов по прямому адресу или из регистра.
Asm\AsmHelper64.cpp — AsmHelper64.cpp — Реализация AsmHelper64: x64 пролог сохраняет rcx/rdx/r8/r9 в shadow space, GenCall резервирует выровненный по 16 байт стек и расставляет до 4 аргументов в регистрах, остальные — на стеке.
Asm\LDasm.c — LDasm.c — Реализация length-disassembler-а LDasm (на основе таблиц флагов опкодов flags_table), вычисляющая длину одной x86/x64-инструкции с учётом префиксов, REX, ModRM, SIB, displacement и immediate полей; используется для патчинга и снятия хуков.

=== DriverControl ===
DriverControl\DriverControl.h — DriverControl.h — Класс DriverControl (синглтон) — клиент для взаимодействия с сопутствующим kernel-mode драйвером BlackBone: загрузка/выгрузка драйвера, выделение/чтение/запись памяти процесса через драйвер, инъекция и ручной маппинг DLL в режиме ядра, изменение защиты процесса (ProtectProcess), сокрытие VAD-регионов (ConcealVAD), отсоединение таблицы хэндлов (UnlinkHandleTable).
DriverControl\DriverControl.cpp — DriverControl.cpp — Реализация DriverControl: EnsureLoaded/Reload открывают хэндл устройства \\.\BlackBone или регистрируют и запускают службу драйвера (выбор файла драйвера в зависимости от версии Windows — 7/8/8.1/10), Unload останавливает службу.

Include\NativeStructures.h — NativeStructures.h — Шаблонные (по разрядности T) определения внутренних NT-структур: _LIST_ENTRY_T, _UNICODE_STRING_T, _NT_TIB_T, _CLIENT_ID_T, _GDI_TEB_BATCH_T, _ACTIVATION_CONTEXT_STACK_T и др., позволяющие работать с 32- и 64-битными представлениями PEB/TEB одинаковым кодом.

=== LocalHook ===
LocalHook\HookHandlerCdecl.h — HookHandlerCdecl.h — Специализация шаблона HookHandler для функций с конвенцией вызова __cdecl: статический Handler извлекает указатель на инстанс хука через TEB (ArbitraryUserPointer) и вызывает CallOriginal/CallCallback в нужном порядке (HookFirst/HookLast/NoOriginal).
LocalHook\HookHandlerFastcall.h — HookHandlerFastcall.h — Специализация HookHandler для функций __fastcall, по аналогии с cdecl-версией формирует обёртку-перехватчик с тем же механизмом получения указателя инстанса через TEB.
LocalHook\HookHandlers.h — HookHandlers.h — Заглушка-объединитель: объявляет шаблон HookHandler и класс-маркер NoClass, подключает все специализации (Cdecl всегда, Stdcall/Thiscall/Fastcall только в x86-сборке через USE64).
LocalHook\HookHandlerStdcall.h — HookHandlerStdcall.h — Специализация HookHandler для __stdcall-функций с тем же шаблоном Handler/HandlerP/CallOriginal/CallCallback, как у cdecl- и fastcall-версий.
LocalHook\HookHandlerThiscall.h — HookHandlerThiscall.h — Специализация HookHandler для __thiscall-функций (методов классов C++ в x86), вызывающая оригинал через приведение типа typeR с учётом скрытого this-аргумента в ecx.
LocalHook\LocalHookBase.h — LocalHookBase.h — Класс DetourBase — базовая инфраструктура локального (внутрипроцессного) хука: перечисления CallOrder/HookType/ReturnMethod, выделение буфера переходника рядом с целью (AllocateBuffer), включение/отключение хука (EnableHook/DisableHook), переключение аппаратных breakpoint-ов (ToggleHBP) и векторные обработчики исключений (VectoredHandler, Int3Handler, AVHandler, StepHandler).
LocalHook\TraceHook.h — TraceHook.h — Класс TraceHook (синглтон) — система трассировочных хуков через исключения: HookContext хранит состояние трассировки указателя (TS_Step/TS_StepInto/TS_WaitReturn), ApplyHook/RemoveHook регистрируют перехват вызовов через подмену значения указателя на адрес-ловушку и обработку через VEH.
LocalHook\LocalHook.hpp — LocalHook.hpp — Шаблонный класс Detour<Fn, C> (наследник HookHandler) — высокоуровневый API локального хука: метод Hook() выбирает реализацию (HookInline/HookInt3/HookHWBP) в зависимости от HookType, Restore() возвращает исходный код функции.
LocalHook\VTableHook.hpp — VTableHook.hpp — Шаблонный класс VTableDetour<Fn, C> (наследник Detour) — хук виртуальной функции через подмену указателя в VTable объекта; умеет копировать всю VTable (copyVtable) перед модификацией конкретного слота index.
LocalHook\LocalHookBase.cpp — LocalHookBase.cpp — Реализация DetourBase: AllocateBuffer ищет свободный регион памяти как можно ближе к цели (для относительных переходов в пределах +-2ГБ) и резервирует страницу PAGE_EXECUTE_READWRITE; DisableHook/EnableHook переписывают первые байты функции или переключают аппаратные точки останова через регистр Dr7.
LocalHook\TraceHook.cpp — TraceHook.cpp — Реализация TraceHook на базе BeaEngine-дизассемблера: ApplyHook создаёт/обновляет HookContext для конкретного указателя, подставляя в него специальное ловушечное значение, отлавливаемое векторным обработчиком исключений VecHandler для определения момента вызова через указатель.

=== ManualMap ===
ManualMap\MExcept.h — MExcept.h — Класс MExcept — поддержка SEH/C++ исключений для вручную замапленного (manual mapped) кода: CreateVEH внедряет в процесс обёртку Vectored Exception Handler и таблицу диапазонов модулей (ModuleTable) для x64, чтобы исключения корректно обрабатывались вне нормального загрузчика.
ManualMap\MMap.h — MMap.h — Класс MMap — основной интерфейс ручного маппинга: CustomArgs_t собирает произвольные пользовательские аргументы для инициализирующей функции, eLoadFlags задаёт опции загрузки (ManualImports, CreateLdrRef, WipeHeader, HideVAD, MapInHighMem и др.), методы MapImage загружают образ из файла или из буфера памяти в целевой процесс.
ManualMap\Native\NtLoader.h — NtLoader.h — Класс NtLdr — интеграция вручную замапленного модуля с нативным загрузчиком NTDLL: CreateNTReference регистрирует модуль в LdrpHashTable/LdrpModuleIndex/InMemoryOrderModuleList, AddStaticTLSEntry создаёт статический TLS-слот, InsertInvertedFunctionTable добавляет фиктивную запись SAFESEH, Unlink убирает модуль из списков загрузчика.
ManualMap\MExcept.cpp — MExcept.cpp — Реализация MExcept: содержит закомментированный псевдокод оригинального ассемблерного VectoredHandler, который патчит LdrpInvertedFunctionTable для регистрации SEH-обработчиков модулей, размещённых вне обычной загрузки.
ManualMap\MMap.cpp — MMap.cpp — Класс MMap — реализация ручного маппинга (manual mapping) PE-образов в адресное пространство целевого процесса без вызова LoadLibrary: MapImage/MapImageInternal проверяют, не загружен ли модуль уже, копируют образ, выполняют релокации, разрешают импорты, обрабатывают TLS и вызывают точку входа.
ManualMap\Native\NtLoader.cpp — NtLoader.cpp — Реализация NtLdr::Init — определение версии загрузчика по версии ОС (Win7/Win8+), поиск внутренних структур LdrpHashTable/LdrpModuleIndexBase/LdrHeap через сигнатурный поиск (для разных битностей x86/x64) с целью корректной регистрации вручную замапленных модулей.

Misc\Utils.h — Utils.h — Класс Utils — статические утилиты: конвертация строк UTF8/ANSI/Wide (UTF8ToWstring, AnsiToWstring и обратные), FormatString (printf-подобное форматирование), StripPath/GetParent для работы с путями, GetExeDirectory, ToLower, генерация случайных строк.
Misc\InitOnce.cpp — InitOnce.cpp — Класс InitOnce — единоразовая (через _InterlockedCompareExchange) инициализация процесса: получение привилегий SeDebugPrivilege/SeLoadDriverPrivilege, загрузка динамических импортов LoadFuncs, загрузка таблицы символов SymbolLoader, инициализация NameResolve.

Misc\Utils.cpp — Utils.cpp — Реализация Utils: UTF8ToWstring/WstringToUTF8 через MultiByteToWideChar/WideCharToMultiByte с кодовой страницей CP_UTF8, AnsiToWstring/WstringToAnsi с произвольной локалью, FormatString и работа с путями файлов.
Misc\Thunk.hpp — Thunk.hpp — Шаблонный класс Win32Thunk<fn,C> — генератор бинарного thunk-кода (структура ThunkData с захардкоженными машинными инструкциями mov/jmp), позволяющего вызывать метод произвольного C++-класса через простой function pointer, используя TEB/TIB для передачи указателя на инстанс.
Misc\Trace.hpp — Trace.hpp — Заголовок логирования библиотеки: DoTrace/DoTraceV форматируют строку с префиксом "BlackBone: " и выводят через OutputDebugStringA/W (и опционально printf), макрос BLACKBONE_TRACE используется по всему коду для отладочной диагностики (отключается через BLACKBONE_NO_TRACE).

=== Patterns ===
Patterns\PatternSearch.h — PatternSearch.h — Класс PatternSearch — поиск байтовых сигнатур в памяти: конструкторы принимают паттерн как вектор байт, строку или initializer_list; SearchWithHandler ищет с поддержкой wildcard-байта (наивный/std::search), второй метод использует алгоритм Бойера-Мура-Хорспула для точного совпадения без wildcard, SearchRemoteWithHandler работает с удалённым процессом.
Patterns\PatternSearch.cpp — PatternSearch.cpp — Реализация PatternSearch: SearchWithHandler с wildcard использует std::search с компаратором, пропускающим байт-джокер; логика поддерживает logAlignment для ускоренного поиска с заданным выравниванием адресов.

=== PE ===
PE\ImageNET.h — ImageNET.h — Класс ImageNET — парсер метаданных .NET-сборок через COM-интерфейсы (IMetaDataDispenserEx/IMetaDataImport/IMetaDataAssemblyImport): Init открывает образ, Parse извлекает таблицу методов (имя класса+метод -> RVA), GetImageRuntimeVer определяет версию CLR образа.
PE\PEImage.h — PEImage.h — Класс PEImage — основной разборщик формата PE (как для x86, так и x64 образов): хранит структуры импортов (ImportData), экспортов (ExportData), релокаций (RelocData) и секций, предоставляет загрузку образа из файла или памяти и доступ к директориям PE-заголовка.
PE\ImageNET.cpp — ImageNET.cpp — Реализация ImageNET::Init/Parse: создаёт COM-объект CLSID_CorMetaDataDispenser, открывает scope метаданных образа (с обходом ошибки CLDB_E_BADUPDATEMODE через установку режима MDUpdateIncremental) для последующего извлечения информации о методах .NET-сборки.
PE\PEImage.cpp — PEImage.cpp — Реализация PEImage::Load — загрузка PE-файла как образа (SEC_IMAGE) либо как простых данных через CreateFileMapping/MapViewOfFile, разбор заголовков (Parse) и подготовка контекста активации (PrepareACTX) для манифестов/SxS.

=== Process ===
Process\MemBlock.h — MemBlock.h — Класс MemBlock и его pImpl MemBlockImpl — RAII-обёртка над блоком памяти удалённого процесса: хранит адрес/размер/защиту, опционально владеет блоком и автоматически освобождает его в деструкторе (Free), также содержит хелпер CastProtection для коррекции флагов защиты при отсутствии DEP.
Process\Process.h — Process.h — Класс Process — главный фасад библиотеки, объединяющий все подсистемы: ProcessCore (хэндл/PID), ProcessMemory, ProcessModules, ProcessThreads, RemoteExec/RemoteHook/RemoteLocalHook, NtLdr и MMap; методы Attach/CreateAndAttach подключаются к существующему или новому процессу.
Process\ProcessCore.h — ProcessCore.h — Класс ProcessCore — низкоуровневое ядро: хранит хэндл и PID процесса, состояние WOW64-барьера (isWow64), указатель на Native (платформенно-специфичные системные вызовы), доступ к PEB32/PEB64 и проверку защищённости процесса (isProtected).
Process\ProcessMemory.h — ProcessMemory.h — Класс ProcessMemory (наследник RemoteMemory) — операции с памятью целевого процесса: Allocate/AllocateClosest (выделение около заданного адреса), Free, Query (информация о регионе), Protect (изменение защиты), Read/Write (в том числе по цепочке указателей через adrList).
Process\ProcessModules.h — ProcessModules.h — Класс ProcessModules — перечисление и поиск загруженных модулей целевого процесса: GetModule по имени/базовому адресу с учётом типа (32/64 бит) и метода поиска (LdrList/Sections/PEHeaders), структура exportData описывает разрешённый экспорт (включая форвардинг на другую DLL).
Process\MultPtr.hpp — MultPtr.hpp — Шаблонный класс multi_ptr<T> — обёртка многоуровневого указателя (цепочка офсетов база+offset1+offset2+...) для локального процесса, безопасно разыменовывающая цепочку через SEH (__try/__except) и возвращающая итоговый типизированный указатель.
Process\RPC\RemoteContext.hpp — RemoteContext.hpp — Класс RemoteContext — контекст удалённого вызова при срабатывании хука: предоставляет доступ к ProcessMemory, потоку (Thread), нативному CONTEXT64 регистров и операциям с адресом возврата текущего стек-фрейма (returnAddress).
Process\RPC\RemoteFunction.hpp — RemoteFunction.hpp — Шаблонный класс RemoteFunctionBase<Conv,R,Args...> — типобезопасный вызов произвольной функции в удалённом процессе: CallArguments оборачивает аргументы в AsmVariant, Call создаёт RPC-окружение и выполняет вызов через ассемблерный генератор, возвращая call_result_t<R>.
Process\RPC\RemoteExec.h — RemoteExec.h — Класс RemoteExec — выполнение произвольного кода в удалённом процессе (Remote Procedure Call): CreateRPCEnvironment подготавливает регион _userData (внутренний результат, результат, статус, событие, аргументы), ExecInNewThread/ExecInWorkerThread/ExecInAnyThread запускают код в новом, выделенном воркер-потоке или в существующем потоке, PrepareCallAssembly генерирует код вызова.
Process\RPC\RemoteHook.h — RemoteHook.h — Класс RemoteHook — установка хуков в удалённом процессе через int3-breakpoint или аппаратные точки останова (HWBP): HookData хранит callback (свободная функция либо метод класса), события onExecute/onReturn, EnsureDebug настраивает дебаг-сессию для перехвата событий останова.
Process\RPC\RemoteLocalHook.h — RemoteLocalHook.h — Класс RemoteLocalHook — установка inline-хука непосредственно в памяти удалённого процесса (без отладочных событий): HookCtx хранит оригинальный и патченый код, размер JMP-перехода, SetHook записывает код-перехватчик, PrepareHook подготавливает буфер около целевого адреса.
Process\RPC\RemoteMemory.h — RemoteMemory.h — Класс RemoteMemory — отображение памяти целевого процесса в текущий через драйвер (Map/Unmap, как для всего процесса, так и для отдельного региона), HookData описывает структуры для перехвата NtAllocateVirtualMemory/NtFreeVirtualMemory/NtMapViewOfSection/NtUnmapViewOfSection в целевом процессе.
Process\MemBlock.cpp — MemBlock.cpp — Реализация конструкторов MemBlock/MemBlockImpl — инициализация полей блока памяти (адрес, размер, защита, владение, физическая память), используется фабричными методами Allocate/AllocateClosest.
Process\Process.cpp — Process.cpp — Реализация класса Process: конструктор инициализирует все подсистемы (core, modules, memory, threads, hooks, remote, mmap, nativeLdr) и вызывает InitializeOnce; Attach подключается к процессу по PID/хэндлу/имени, CreateAndAttach создаёт новый процесс (опционально в приостановленном состоянии).
Process\ProcessCore.cpp — ProcessCore.cpp — Реализация ProcessCore::Open — открытие хэндла процесса (с особым случаем для текущего процесса и Windows 10, где требуется PROCESS_ALL_ACCESS вместо pseudo-хэндла), инициализация подсистемы Native (определение WOW64-барьера).
Process\ProcessMemory.cpp — ProcessMemory.cpp — Реализация ProcessMemory: Allocate/AllocateClosest делегируют в статические методы MemBlock, Free вызывает Native::VirtualFreeExT, с трассировкой операций освобождения/декоммита через BLACKBONE_TRACE.
Process\ProcessModules.cpp — ProcessModules.cpp — Реализация ProcessModules::GetModule — поиск модуля по имени (со снятием пути через Utils::StripPath) или по базовому адресу, с использованием системы резолвинга имён NameResolve и кэша символов SymbolData.
Process\RPC\RemoteExec.cpp — RemoteExec.cpp — Реализация RemoteExec: ExecInNewThread копирует код в целевой процесс (CopyCode) и определяет необходимость переключения режима WOW64->x64 (switchMode) на основе типа барьера wow_32_64, после чего создаёт поток и ждёт завершения.
Process\RPC\RemoteHook.cpp — RemoteHook.cpp — Реализация RemoteHook::EnsureDebug — проверка возможности отладки целевого процесса (несовместимость x64-из-x86, уже работающий дебаггер), создание выделенного потока обработки debug-событий (EventThreadWrap) для перехвата исключений останова.
Process\RPC\RemoteLocalHook.cpp — RemoteLocalHook.cpp — Реализация RemoteLocalHook: AllocateMem резервирует исполняемую память около целевого адреса для размещения thunk-кода, PrepareHook готовит копию оригинального кода и джамп-код перехода на хук с использованием LDasm для определения границ инструкций.
Process\RPC\RemoteMemory.cpp — RemoteMemory.cpp — Реализация RemoteMemory::Map — создаёт именованный pipe для передачи hook-данных и вызывает Driver().MapMemory через драйвер для отображения всего адресного пространства целевого процесса (или отдельного региона) в текущий процесс.

=== Process/Threads ===
Process\Threads\Thread.h — Thread.h — Класс Thread — обёртка над хэндлом потока процесса: операции Suspend/Resume, получение/установка контекста (включая аппаратные регистры Dr0-Dr7 для HWBP — структура regDR7), доступ к TEB (32/64-битному) через метод teb().
Process\Threads\Threads.h — Threads.h — Класс ProcessThreads — управление набором потоков процесса: CreateNew создаёт новый поток в целевом процессе с заданным флагом (CreateSuspended/NoDllCallbacks/HideFromDebug), getAll/getMain/getLeastExecuted/getMostExecuted/getRandom/get(id) — методы выборки потоков по различным критериям.
Process\Threads\Thread.cpp — Thread.cpp — Реализация Thread: конструкторы открывают хэндл по ID или принимают существующий хэндл, teb() делегирует получение TEB в Native, Suspend защищает от дедлока (не приостанавливает текущий поток) и учитывает WOW64-барьер.
Process\Threads\Threads.cpp — Threads.cpp — Реализация ProcessThreads::CreateNew — создание удалённого потока через Native::CreateRemoteThreadT с понижением требуемых прав доступа и, при необходимости, повышением прав хэндла потока через драйвер (Driver().PromoteHandle); getAll использует Toolhelp32-снимок потоков.

=== Subsystem ===
Subsystem\NativeSubsystem.h — NativeSubsystem.h — Класс Native — базовый слой системных вызовов для управления памятью/потоками целевого процесса (VirtualAllocExT, VirtualFreeExT, VirtualProtectExT, ReadProcessMemoryT, WriteProcessMemoryT, VirtualQueryExT), инициализирует Wow64Barrier по комбинации архитектур текущего/целевого процессов.
Subsystem\Wow64Subsystem.h — Wow64Subsystem.h — Класс NativeWow64 (наследник Native) — переопределяет операции с памятью для случая управления 64-битным процессом из 32-битного (WOW64) с использованием X64Call/расширений wow64ext для вызова нативных x64 NTDLL-функций.
Subsystem\x86Subsystem.h — x86Subsystem.h — Класс x86Native (наследник Native) — реализация для чисто 32-битной ОС (x86OS): прямой вызов NtQueryVirtualMemory, GetThreadContextT/SetThreadContextT для WOW64/native контекстов, доступ к PEB32/PEB64/TEB32/TEB64.
Subsystem\NativeSubsystem.cpp — NativeSubsystem.cpp — Реализация конструктора Native — определяет тип WOW64-барьера (wow_32_32/wow_64_64/wow_32_64/wow_64_32) через сравнение IsWow64Process текущего и целевого процессов, с особым случаем при принудительном режиме x86OS.
Subsystem\Wow64Subsystem.cpp — Wow64Subsystem.cpp — Реализация NativeWow64::VirtualAllocExT/VirtualFreeExT — получение адреса функции NtAllocateVirtualMemory/NtFreeVirtualMemory из 64-битного NTDLL через GetProcAddress64 и вызов её через X64Call (переключение в long mode для вызова x64-кода из WOW64-процесса).
Subsystem\x86Subsystem.cpp — x86Subsystem.cpp — Реализация x86Native::VirtualQueryExT/GetThreadContextT — прямой системный вызов NtQueryVirtualMemory с маппингом результата MEMORY_BASIC_INFORMATION в 64-битную структуру, обёртки над стандартными GetThreadContext/SetThreadContext для чистой x86-системы.

=== Symbols ===
Symbols\PatternLoader.h — PatternLoader.h — Объявление функции ScanSymbolPatterns — сигнатурный (байт-паттерн) поиск внутренних адресов NTDLL (32- и 64-битного образов) для заполнения структуры SymbolData, когда символы PDB недоступны.
Symbols\PDBHelper.h — PDBHelper.h — Класс PDBHelper — обёртка над DIA SDK (Debug Interface Access) для извлечения адресов символов из PDB-файлов: Init загружает символы для указанного PE-файла, GetSymAddress возвращает RVA по имени символа, с кэшированием найденных адресов в unordered_map.
Symbols\SymbolData.h — SymbolData.h — Структура SymbolData — набор адресов внутренних, недокументированных функций/данных NTDLL (LdrpHandleTlsData, LdrpInvertedFunctionTable, RtlInsertInvertedFunctionTable, LdrpReleaseTlsEntry, LdrProtectMrdata, LdrKernel32PatchAddress) для x86 и x64, заполняется через SymbolLoader.
Symbols\SymbolLoader.h — SymbolLoader.h — Класс SymbolLoader — загрузчик карты символов NTDLL: Load пытается получить адреса сначала из PDB (LoadFromSymbols через PDBHelper), затем дополняет недостающие сигнатурным поиском (LoadFromPatterns); LoadImages подгружает x86/x64 образы ntdll.dll с диска.
Symbols\PatternLoader.cpp — PatternLoader.cpp — Реализация ScanSymbolPatterns/FindPattern — поиск адреса по байтовому шаблону (PatternSearch) с последующим вычислением итогового адреса либо как прямого смещения функции, либо через разыменование операнда инструкции (для разных битностей образа).
Symbols\PDBHelper.cpp — PDBHelper.cpp — Реализация PDBHelper::Init/CoCreateDiaDataSource — настройка пути символьного кэша (_NT_SYMBOL_PATH либо временная папка с публичным сервером Microsoft), создание DIA источника данных и сессии для последующего поиска символов по имени.
Symbols\SymbolData.cpp — SymbolData.cpp — Определение глобальной переменной g_symbols типа SymbolData — единого экземпляра найденных адресов внутренних структур NTDLL, используемого всеми подсистемами библиотеки (NtLdr, MExcept и др.).
Symbols\SymbolLoader.cpp — SymbolLoader.cpp — Реализация SymbolLoader::Load/LoadFromSymbols — определение, требуется ли работа с FsRedirector (для доступа к System32 из WOW64-процесса), загрузка символов 32- и 64-битного ntdll.dll через PDBHelper, с последующим дополнением сигнатурным поиском при неудаче.

=== Syscalls ===
Syscalls\Syscall.h — Syscall.h — Шаблонная функция syscall::syscall<R>(index, args...) — типобезопасная обёртка прямого системного вызова (минуя перехваченные пользовательские хуки NTDLL) через syscall_stub; get_index определяет номер системного вызова по экспортируемой Nt*-функции NTDLL (читая байты её тела).
Syscalls\Syscall32.asm — Syscall32.asm — Ассемблерный модуль (MASM, x86): x64_enter/x64_exit переключают процессор в long-mode и обратно через far-call/far-ret трюк (push 0x33/0x23 + retf), _syscall_stub формирует переход в 64-битный режим для выполнения системного вызова из WOW64-процесса.
Syscalls\Syscall64.asm — Syscall64.asm — Ассемблерный модуль (MASM, x64): syscall_stub — компактная заглушка, которая раскладывает переданные через регистры/стек аргументы по конвенции синтетического вызова (rcx,rdx,r8,r9 + стек) в регистры r10/rdx/r8/r9 согласно соглашению syscall и выполняет инструкцию syscall напрямую, минуя хуки в ntdll.
