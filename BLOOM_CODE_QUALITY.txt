POTENTIAL ISSUES & MITIGATIONS:
================================

Issue 1: GPU Resource Exhaustion
  Symptom: Render() returns false
  Cause: Device lost or out of VRAM
  Mitigation: Check IsInitialized(), retry allocation
  Detection: Monitor GetResult() for nullptr

Issue 2: Shader Compilation Failure
  Symptom: Initialize() returns false
  Cause: Invalid HLSL code or driver issue
  Mitigation: Check NWARN logs, update drivers
  Detection: Use D3D11 debug layer

Issue 3: Viewport State Leakage
  Symptom: Scene viewport wrong after bloom
  Cause: Exception before viewport restore
  Mitigation: Use RAII for viewport stack
  Detection: Verify in post-render checks

Issue 4: Color Space Mismatch
  Symptom: Colors appear wrong
  Cause: Mixing linear/gamma spaces
  Mitigation: Ensure consistent color space
  Detection: Compare with reference images

ARCHITECTURE DECISIONS:
=======================

1. Why constexpr HLSL?
   - Zero runtime parsing overhead
   - Compile-time syntax checking (in IDE)
   - No external shader files needed
   - Simplified deployment

2. Why std::unique_ptr?
   - Automatic resource cleanup
   - Exception-safe (if used correctly)
   - Move semantics support
   - Clear ownership semantics

3. Why dynamic downscaling?
   - User control over quality/speed
   - Adaptive performance available
   - Memory efficiency option
   - Consistent API interface

4. Why separate blur passes?
   - Separable blur reduces complexity
   - Better cache locality
   - Easier to extend (3D blur)
   - Standard industry practice

5. Why limited shader targets?
   - ps_4_0 supports DirectX 10+ hardware
   - vs_4_0 sufficient for fullscreen pass
   - Future expansion to ps_5_0 easy
   - Maintains backward compatibility
